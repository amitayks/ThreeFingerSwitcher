## Why

Two visible defects share one root: the switcher re-derives a window's identity and appearance from a *degraded live view* whenever it cannot observe that window cleanly on the current Space. Off-current-Space, the listing falls to a weak CoreGraphicsServices-metadata gate that surfaces **ghost windows** (background-process windows, dialogs, and dismissed sheets that aren't really open — e.g. ASUS GlideX, and a Terminal dialog/sheet with no Accessibility window, observed as a "terminate running processes?" prompt). And the thumbnail refresh re-captures windows in their current on-screen state, so a **Stage-Manager set-aside** window is captured as the tilted/perspective strip proxy and that degraded image overwrites the previously-good cached thumbnail.

## What Changes

- **Off-Space listing — stop surfacing ghosts.** Require an Accessibility element (live via remote-token brute force, or from the persistent cache populated while the window was reachable) to list an off-Space window in `WindowService.snapshot()`, and **remove the CoreGraphicsServices metadata-only fallback**. The `--diag` dump is decisive: the real ghosts (ASUS GlideX wid 16614, and a Terminal no-AX window wid 1760 observed as a "terminate?" sheet) are the *only* windows with no Accessibility element on any Space (`live=0 cached=0`), while every genuine window — including off-Space Chrome — resolves one; and they are NOT distinguishable by CGS metadata (alpha, size, Space membership all look real). Because requiring an element makes the previously-planned **negative-observation cache** and the **empty-Space gate** (`CGSCopySpacesForWindows`, which returns `1` for every window on Tahoe) dead — they only ever fed the metadata gate — both are removed, net-deleting code rather than adding it (converging on AltTab's 100%-Accessibility model). Genuine off-Space windows, including Chromium windows a brute force can't reach (the "Bug A" case), keep listing via the **cached element**, so cache seeding (on app activation and every current-Space pass) is the load-bearing Bug-A guard and is preserved/strengthened.
- **Thumbnails — trust the last clean capture; revert the hardware-capture experiment.** A mid-change experiment routed Stage-Manager captures through the private window-server hardware capture (`CGSHWCaptureWindowList`, `fullSize`); on Tahoe it returns stale/garbage backing and made P1 *worse* (corrupt/blank, not just tilted), so it is **reverted** (the HW path in `capture()` and `cgs.hwCaptureWindowList` are removed). The remaining, correct mechanism stands: `ThumbnailService.capture()` detects a degraded presentation (set-aside strip proxy / off-screen frame) and bails (keeps the cached image or icon); `prefetch()` is cache-first for windows that are not cleanly visible (off-Space, minimized, or set aside under Stage Manager) and only live-captures cleanly-visible windows; the cache is warmed while a window is cleanly front. Stage Manager set-aside windows come in two presentations: a strip on a non-current Space parks windows off every display (caught by the off-screen test), while a strip on the current Space sits on-screen at positive coordinates with CGWindowList reporting a small scaled rect — so the system carries each window's real (Accessibility) size and skips a capture whose displayed frame is a small fraction of it (a strip thumbnail), serving the cached preview or icon.
- **Diagnostics first.** Extend `diagnosticReport()` (the `--diag` path and the in-app "Write Diagnostics" menu) to dump, per layer-0 regular-app candidate (including dropped ones), its CGS attributes (name, bounds, alpha, on-screen flag) plus whether a live and/or cached Accessibility element resolved and the listing decision, and to dump SCK `scWindow.frame` vs the window's logical frame. (Two columns added during investigation are dropped with the code they probed: CGS Space membership — inert on Tahoe, `spaces=1` for every window — with the empty-Space gate, and the window-server hardware-capture size with the reverted HW-capture binding.) The listing/thumbnail discriminators are then chosen from captured data, not guessed.

No change to gesture recognition, raising/focus, the overlay panel/view, or the legacy current-Space path.

## Capabilities

### New Capabilities

_None._

### Modified Capabilities

- `window-enumeration-and-raising`: the **off-Space enumeration** requirement now requires an Accessibility element (live or cached) to list an off-Space window and drops the CoreGraphicsServices metadata-only fallback, so ghosts with no element on any Space (closed-but-process-alive windows, dialog/sheets, background-agent surfaces) are excluded while genuine off-Space windows — including Chromium reachable only via the cached element — still list; the **off-Space raise** requirement's "no-AX window" scenario is reframed — such a window is no longer listed, so it cannot be committed (the front-the-app-without-switching behavior remains only as a defensive no-crash guarantee for a listed window whose element goes stale at commit); the **thumbnail refresh** requirement keeps its "do not overwrite a clean thumbnail with a degraded capture" guarantee, with the skip discriminator clarified to set-aside/off-every-display/minimized (an off-Space-but-on-screen window still refreshes), not "off the current Space". A new **diagnostics** requirement covers the `--diag` per-candidate listing/thumbnail dump.

## Impact

- `Sources/ThreeFingerSwitcher/Windows/WindowService.swift` — off-Space listing requires a live-or-cached Accessibility element; the metadata-only gate (`metadataListable`), the `negativeSwitchable` cache, and the `CGSCopySpacesForWindows` empty-Space gate are removed as dead code; element-cache seeding (activation + current-Space pass) preserved/strengthened as the Bug-A guard; `diagnosticReport()` keeps the per-candidate dump but is reduced to the AX-required model (the `spaces=`, `neg=`/`negCache=`, `negativeSwitchable set size`, and `metadataListable`-based decision columns are dropped with the code they describe).
- `Sources/ThreeFingerSwitcher/Windows/ThumbnailService.swift` — **revert** the `CGSHWCaptureWindowList` hardware-capture path in `capture()`; keep degraded-capture detection, cache-first `prefetch()`, and optional frame logging on the SCK path; add `isStripProxy(displayedFrame:realFrame:)` so a current-Space Stage-Manager strip thumbnail (CGWindowList frame ≪ Accessibility frame) is skipped.
- `Sources/ThreeFingerSwitcher/Windows/WindowInfo.swift` — add `realFrame` (the Accessibility/real window size) so the thumbnail path can tell a scaled strip proxy from a clean window; populated in `WindowService.snapshot()`.
- `Sources/ThreeFingerSwitcher/Windows/CGSPrivate.swift` — remove the now-unused `hwCaptureWindowList` binding (and, if unused elsewhere, `CGSCopySpacesForWindows`).
- `Sources/ThreeFingerSwitcher/Windows/Spaces.swift` — `spaceCount` becomes unused once the empty-Space gate is removed (CGS Space membership is `1` for every window on Tahoe); drop if dead.
- `Tests/ThreeFingerSwitcherTests/` — degraded-capture decision coverage retained; tests asserting the metadata-only gate / negative-cache behavior are updated to the AX-required model.
- Constraints unchanged: never crash / never regress the legacy current-Space path (`cgs.offSpaceSupported` guard); `SpaceGrouping.group()` stays pure; app is unsandboxed; GPL-3 (AltTab-derived techniques). Separate from the in-progress `disable-spaces-auto-rearrange` and `stable-space-row-order` changes.
