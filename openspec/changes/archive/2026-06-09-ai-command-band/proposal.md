## Why

The launcher already gives us a trackpad-native, keyboardless, non-activating way to scrub a list and commit by dwell-and-lift, while the app the user was in stays frontmost. Clipboard history proved that this surface can host *more than launching* — it can host **verbs that act on what you are looking at right now**. The most valuable such verb is a local LLM. With Gemma 4 (Apache-2.0, runs in-process on Apple Silicon via MLX) a strong Mac can now run a frontier-class model **on-device, for free, fully private, offline** — so "fix this grammar / make this concise / translate / explain / turn this into a calendar event / save this to my project" becomes a single dwell-lift on a curated command, with the model's result **streamed into the preview pane as a review** before it ever touches the user's document. No keyboard, no chat window, no second app, no cloud.

## What Changes

- **A new "AI Command" launcher band** populated by **user-configured commands** (curated prompt presets). Like the clipboard band it is non-activating, fired by the existing dwell-to-arm / lift-to-commit grammar, and works in **any** front app. Unlike clipboard it is a *static* configured band (authored in Settings), not a streamed one.
- **Each command is a value type** — `name, icon, tint, input source, prompt template (with tokens), output target, model, confirmBeforeRun` — authored and persisted in Settings (keyboardless at use time; freeform prompts are written once at config time).
- **Two execution modes from one mechanism:**
  - **In-place edit** — take the front app's **selected text** (or clipboard / screen region), transform it, and **paste/replace it back where the user was**.
  - **Background task (agentic)** — interpret the input and *perform an action* via **schema-targeted structured output** (validated, repaired on mismatch, and **declinable** — the model can say "not applicable" instead of fabricating): add-to-calendar (`EventKit`), save-to-project (on-disk note), open-a-tool-with-this-payload (`LaunchService`), send-to-destination (adapter). Each side-effecting task shows an **action-review preview** before firing **by default** (user-overridable per command).
- **The preview pane becomes a live streaming output canvas.** On lift, the band stays open and the model **streams tokens into the preview**; a fresh four-finger **down swipe commits** (paste / run the task — "bring it into the document"), a **horizontal swipe discards**. This turns model latency into a review and makes "it rewrote my text, undo undo" impossible.
- **A new on-device AI runtime layer.** A `LLMRuntime` seam with one v1 conformer: **Gemma 4 31B** (the flagship — text + vision, max quality) run **in-process via MLX-Swift**, with **streaming generation**, **schema-targeted structured output** for the task modes (validated with repair-on-mismatch, declinable; XGrammar via `mlx-swift-structured` available as an optional technique), **vision** input for screen-region commands, and a **model lifecycle manager** (opt-in download of QAT 4-bit weights, verify, lazy-load, keep-resident, evict on memory pressure). The seam exists so other Gemma 4 sizes (e.g. the audio-capable 12B), a future Gemma, Apple Foundation Models, or a cloud model can drop in later **without touching feature code**; v1 tests only Gemma 4.
- **A new selection-I/O primitive.** Read the front app's current input via Accessibility **`AXSelectedText`** (no clipboard clobber, reusing the Accessibility permission we already hold); fall back to synthesized **⌘C with prior-clipboard restore**; optionally **capture a screen region** (Screen Recording is already held) for vision commands. Write results back via **`AXSelectedText` set when settable, else paste** (reusing `LaunchService` paste-on-fire).
- **Targets the strongest Apple Silicon only.** We use the best model the hardware allows (M4/M5-class, ample unified memory); there is **no low-end-Mac fallback** in v1. Quality and correctness are prioritized over speed and footprint.
- **Settings surface:** an "AI commands" opt-in (default OFF) that gates the band and unlocks the model download; the command editor (authoring name/prompt/input/output/model/confirm, reorder, delete); model-management UI (download/status/evict); and per-task consent (Calendar permission).

## Capabilities

### New Capabilities
- `ai-command-band`: the configurable band of named AI commands — the `AICommand` value model and its on-disk persistence, the keyboardless **authoring UI** in Settings (prompt templates with `{input}`/`{date}`/`{app}`/`{url}` tokens, input source, output target, model, confirm), the **synthetic band built fresh on launcher open** from the configured commands, dwell/lift to fire, and the **end-to-end execution orchestration** (gather input → render template → run model → route output) for the **in-place edit** mode (replace selection / paste at cursor / preview-only).
- `on-device-ai-runtime`: the swappable model layer — the `LLMRuntime` abstraction (capabilities, streaming generation, schema-targeted structured output, vision), the **Gemma 4 via MLX-Swift** conformer (31B default; capability-based size selection for later audio/speed variants), **schema-targeted structured output with validation and repair** for task JSON (no hard token-caging; the model may decline), and the **model lifecycle** (opt-in download of QAT 4-bit weights with integrity verification, lazy load, keep-resident-between-calls, evict on memory pressure, a model registry of id→size/sha/url/capabilities).
- `selection-io`: capturing the front app's current input and returning output to it — AX **`AXSelectedText`/`AXSelectedTextRange`** read, **⌘C-with-restore** fallback, **screen-region capture** for vision; and output write-back via **AX selection replace when settable, else paste at cursor** — all reusing the already-held Accessibility/Screen-Recording permissions, with no clipboard clobber on the happy path.
- `ai-command-tasks`: the agentic background-task modes driven by **schema-targeted structured output** (validated, repairable, and declinable), each with an **action-review preview** before side effects — **add-to-calendar** (`EventKit`, consent-gated), **save-to-project** (on-disk note reusing the clipboard store pattern), **open-tool-with-payload** (`LaunchService` + generated payload file / Shortcut), and **send-to-destination** (adapter: Shortcut / URL scheme / shell-out). `confirmBeforeRun` **defaults on** for side-effecting tasks but **honors the user's stored value**.

### Modified Capabilities
- `launcher-overlay`: introduces a **live streaming output canvas** preview mode — distinct from clipboard's static master-detail — where, after lifting on an AI command, the overlay **stays open** and streams model tokens into the preview pane, then resolves via a fresh four-finger **down swipe = commit** or **horizontal swipe = discard**; AI command items render in the icon grid; and a side-effecting task can enter a distinct **armed-confirmation ("review the action") state** before it fires.
- `tunable-settings`: a new **"AI commands" opt-in** (default OFF) that gates the band and the model download; storage and editor surface for the **command list**; **model-management** controls (which Gemma 4 model, download/status/size, evict, residency); older settings load unchanged with the feature off.
- `permissions-onboarding`: adds the **Calendar (`EventKit`) permission**, requested **only** when a calendar task first runs (consent-gated, never at launch); and documents the **reuse** of the already-held Accessibility (selected-text read/replace) and Screen Recording (screen-region vision) permissions — no new entitlement beyond Calendar, App Sandbox stays off.

## Impact

- **New (model runtime):** `Sources/ThreeFingerSwitcher/AI/LLMRuntime.swift` (protocol + request/response/capability types), `AI/GemmaMLXRuntime.swift` (Gemma 4 via `gemma-4-swift-mlx` + `mlx-swift-structured`), `AI/ModelManager.swift` (registry, download, verify, residency, eviction), `AI/StructuredSchema.swift` (task JSON schemas). New SwiftPM deps: `VincentGourbin/gemma-4-swift-mlx`, `ml-explore/mlx-swift-lm`, `petrukha-ivan/mlx-swift-structured` (transitively MLX/Metal — **requires `xcodebuild`, not `swift build`**, for the runtime target).
- **New (selection I/O):** `Sources/ThreeFingerSwitcher/AI/SelectionService.swift` (AX read/replace, ⌘C-restore fallback, screen-region capture), extending the existing `AXPrivate.swift` helpers.
- **New (band + commands):** `AI/AICommand.swift` (value model + input/output/task enums), `AI/AICommandStore.swift` (persistence), `AI/AICommandBandBuilder.swift` (store → synthetic band, mirrors `ClipboardBandBuilder`), `AI/AICommandExecutor.swift` (input→template→model→output orchestration), `Settings/AICommandEditorView.swift` (authoring), and a streaming preview-canvas view.
- **New (tasks):** `AI/Tasks/*` — `CalendarTask` (EventKit), `SaveToProjectTask`, `OpenToolTask`, `SendToTask`, plus a `TaskDispatcher` + confirmation-preview model.
- **Modified:** `Overlay/LauncherModel.swift` + `Overlay/LauncherOverlayController.swift` + `Overlay/LauncherView.swift` + `Gesture/GestureRecognizer.swift` + `App/AppCoordinator.swift` (AI band kind, streaming canvas, swipe-to-resolve: down-swipe-commit / horizontal-swipe-discard via a one-shot canvas-resolution mode, armed-confirmation state), `Launcher/LaunchItem.swift` (a synthetic `.aiCommand` cell, non-persisted in Favorites), `Launcher/LaunchService.swift` (paste/replace into `capturedFrontApp`, open-tool payload), `App/AppCoordinator.swift` (inject the AI band on open; front-app capture already exists), `Settings/AppSettings.swift` + `Settings/SettingsView.swift` (opt-in, command list, model management), `PermissionsService.swift` (EventKit check/request).
- **Permissions:** **new = Calendar (`EventKit`)**, consent-gated at first calendar-task use. Selected-text read/replace reuses **Accessibility** (already held); screen-region vision reuses **Screen Recording** (already held). App Sandbox stays off; no native-gesture relocation, no re-login.
- **Privacy:** selected text and screen regions are sent to a **local, on-device** model only (on-brand). Any cloud command (later) is a per-command, clearly-labeled consent gate. Save-to-project writes user content to a local file (user-controlled). First-run downloads multi-GB model weights (opt-in, the reason the feature is default OFF).
- **Build/verify:** the MLX runtime target compiles only under `xcodebuild` (Metal); the rest of the feature stays behind `LLMRuntime` with a **stub conformer** so it still verifies under the project's `swift build` / `swift test` rule (CLAUDE.md).
