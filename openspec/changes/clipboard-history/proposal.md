## Why

People keep a clipboard history in a separate tool (Raycast, Maccy, Flycut) just to paste something they copied a few minutes ago. The launcher already gives us a trackpad-native way to scrub a list and commit by dwell-and-lift — so the history can live *inside* the launcher as one more band, reusing muscle memory the user already has, with no keyboard, no focus-stealing window, and no second app. Because the launcher overlay is non-activating, the app the user copied from stays frontmost the whole time, so "pick an entry, paste it where I was" falls out for free.

## What Changes

- **A background clipboard recorder (opt-in, default OFF).** When enabled, the app polls `NSPasteboard.general.changeCount` and snapshots each new copy — capturing **all useful representations** (rich text + plain, image bytes, file-URL references, color, URL) so a later paste is byte-faithful. It honors `org.nspasteboard.ConcealedType` / `TransientType` (password managers), a per-app exclusion list, a global pause, and "clear history". History is stored **on disk, separate from Favorites** (text inline, images/large payloads as blobs), with de-dup and retention caps by count, total bytes, and age.
- **A synthetic "Clipboard" band in the launcher**, present only while the opt-in is on, always the **last** band. It is **built fresh on every launcher open** from the store (a recent-window slice, pinned entries first) and is **never written into the Favorites record**. It renders as a **master-detail**: a multi-line list of truncated **keys** on the left, and a large **value preview** on the right that shows the *actual content* — rendered image, QuickLook content preview for files, full text, color swatch — not just a type icon.
- **Navigation reuses the existing grammar, with band-specific horizontal semantics.** Vertical scrubs the key list (dwell-to-arm, lift-to-paste, exactly like firing an item). Horizontal does **not** cross into the value column (cut by design). Instead, on a selected entry: **swipe RIGHT pins it**, **swipe LEFT moves to the previous band** (fast exit, since Clipboard is last). A pin does **not** reorder the live list mid-session — the pinned-first ordering is applied on the **next** band build (pin-now-use-later).
- **Edge-triggered scroll acceleration.** When scrubbing a long list and the controlling contact reaches the trackpad/overlay edge, the list keeps advancing on its own and **accelerates** the longer the edge is held — so a long history is traversable without repeated lift-reposition-scrub (the chief cost of having no keyboard search).
- **Paste on fire.** Lifting on an armed entry restores that entry's representations to `NSPasteboard.general` and pastes into the app that was frontmost when the launcher opened (synthesized ⌘V), using **only the Accessibility permission the app already holds** — no new permission, no new entitlement.
- **Settings surface:** a "Keep clipboard history" toggle (default OFF) that gates both the recorder and the band, plus tunables (recent-window size, retention caps, poll interval, edge-acceleration sensitivity), a clear-history action, and the app-exclusion list.

## Capabilities

### New Capabilities
- `clipboard-history`: the opt-in recorder and on-disk store (multi-representation capture, privacy/concealed handling, de-dup, retention caps, versioned persistence); the synthetic, non-persisted Clipboard band built from the store (recent window, pinned-first); the pin model (right-swipe toggle, deferred reorder); and paste-on-fire into the captured front app.

### Modified Capabilities
- `launcher-overlay`: the Clipboard band introduces a **master-detail layout mode** (key list + value preview) distinct from the icon grid; its **horizontal travel is repurposed** (RIGHT pins the selected entry, LEFT switches to the previous band) rather than stepping items or crossing columns; and a new **edge-triggered scroll acceleration** applies when navigating an overflowing list.
- `tunable-settings`: a new **clipboard-history opt-in** (default OFF) that gates the recorder and the band, with tunables (recent-window size, retention caps, poll interval, edge-acceleration sensitivity) and settings-UI controls for clearing history and managing the app-exclusion list; older settings load unchanged with the feature off.

## Impact

- **New:** `Sources/ThreeFingerSwitcher/Clipboard/ClipboardEntry.swift` (value model + representations), `ClipboardStore.swift` (on-disk index + blobs, retention, de-dup, pins), `ClipboardMonitor.swift` (changeCount poll + capture + concealed-skip), `ClipboardBandBuilder.swift` (store → synthetic band), and a value-preview view (image / QuickLook / text / color).
- **Modified:** `Overlay/LauncherModel.swift` (clipboard band kind + vertical-scrub + horizontal pin/previous-band), `Overlay/LauncherOverlayController.swift` (master-detail sizing, band injection at `show`, edge auto-scroll timer), `Overlay/LauncherView.swift` (render the clipboard band), `Launcher/LaunchService.swift` or a sibling (paste-on-fire: restore reprs + ⌘V into `capturedFrontApp`), `Launcher/LaunchItem.swift` (a synthetic, non-persisted clipboard-entry cell), `App/AppCoordinator.swift` (inject the band at open; the front-app capture already exists), `Settings/AppSettings.swift` + `Settings/SettingsView.swift` (toggle, tunables, clear, exclusions), and `Gesture/GestureRecognizer.swift` / `TouchInput/TouchEngine.swift` (edge detection from normalized contact position).
- **Permissions:** none new — pasting uses the already-held Accessibility; reading the general pasteboard needs none. **App Sandbox stays off** (unchanged). No native-gesture relocation, no re-login.
- **Privacy:** introduces persistent storage of copied content (the reason it is strictly opt-in and default OFF); concealed/transient items and excluded apps are never recorded; the store is local-only and user-clearable.
