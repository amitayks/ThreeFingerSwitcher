## Why

Enabling AI commands with no internet exposed two real bugs and a systemic gap behind them:

1. **The Settings window freezes.** The "Couldn't prepare the model" alert is an **application-modal `NSAlert.runModal()`** (`AppCoordinator.swift:1123`), presented free-standing (never `beginSheetModal(for:)`). Its nested run loop suspends event delivery to the Settings window, so scrolling and clicking die — only the WindowServer-handled traffic lights survive. (Verified: the freeze is bounded to the alert's lifetime; clicking OK restores input — but the experience reads as "the window is broken.")
2. **The status row shows a raw NSError dump.** `ModelManager.swift:170` builds `.failed(reason: "Failed to provision …: \(error)")` — `\(error)` interpolates the entire wrapped `NSURLError` (Domain/Code/UserInfo/NSUnderlyingError), and `ModelManagementView.swift:63` renders it in a `Text(reason)` with **no line cap, in a fixed-size 460×460 `Form` with no `ScrollView`**, which overflows the frame.

A read-only audit of all 48 error sites across the AI feature found these are not isolated — they are one **systemic pattern**: raw errors interpolated into UI strings (`\(error)` / `String(describing:)`); `RuntimeError` not conforming to `LocalizedError` (so the "clean" path falls back to raw); **four near-duplicate error→string mappers** that disagree (hence the two inconsistent surfaces); raw vendor/OS errors carried across layer boundaries (`GemmaMLXRuntime.prepare` re-throws the raw `Gemma4DownloadError`); a **missing generic catch** on the byte-download path (a non-cancel error leaves the state stuck at `.downloading` forever); and several **silent-success paths** that report a false "Done" (`NSWorkspace.open` fire-and-forget, ignored `Process` termination status, `replaceSelection`/`pasteAtCursor` reporting *attempted* not *landed*, swallowed AX/screen-capture failures). The user asked to fix this everywhere, not just the download — this change does that.

## What Changes

- **One error taxonomy + one translator.** Extend the existing `RuntimeError` with the missing provision/network cases, make it conform to `LocalizedError`, and introduce a single central translator `AIError.message(for:) -> AIPresentedError` (a `headline` plus optional copyable `details`). All current mappers (`AICommandExecutor.message(for:)`, `TaskDispatcher.message(for:)`, `AppCoordinator`'s inline `?? "\(error)"`, and `ModelManager`'s raw interpolation) collapse onto it, so every surface shows the **same clean string**.
- **Map at layer boundaries.** Raw vendor/OS errors (`Gemma4DownloadError`, `NSURLError`, EventKit, `FileManager`/`FileHandle`, `Process`) are converted to the taxonomy where they cross into app code (notably `GemmaMLXRuntime.prepare`), and never propagated raw to a view model or the UI.
- **Ban raw interpolation in UI-bound strings.** `\(error)` / `String(describing:)` / OS `.localizedDescription` are allowed only in logs and as opt-in `details`, never as a headline.
- **Failure is a first-class observable state, not a thrown error left for the caller to surface.** Keep the existing `.failed`-state pattern; the state carries the clean headline. Add the missing generic catch so the state never gets stuck mid-flight.
- **Non-blocking, bounded error UI.** The Settings model-status row renders a concise headline (capped + truncating) with raw details behind a "Show details / Copy" disclosure; the `Form` becomes scroll-safe so unexpected length degrades to scrolling, never a frozen window. Settings-scoped failures stop using an app-modal `NSAlert.runModal()` as their surface (the in-window `.failed` row + Retry is the primary surface).
- **Failure ≠ silence.** The silent-success paths are made honest: a write/open/task that didn't land becomes a `.failed` state with a clean message, not a false "Done."

## Capabilities

### New Capabilities
- `ai-error-handling`: how the AI feature classifies, translates, and surfaces errors end-to-end — the shared error taxonomy, the single error→message translator (clean `headline` + opt-in `details`), the layer-boundary mapping rule, the clean-message invariant for every user-facing surface, the non-blocking + bounded error-rendering rules for Settings and the overlay canvas, and the "failure is never silent" guarantee for side-effecting/selection paths.

## Impact

- **Modified (translator + taxonomy):** `Sources/ThreeFingerSwitcher/AI/LLMRuntime.swift` (extend `RuntimeError`; add cases `offline`, `serverUnavailable`, `authOrAccessDenied`, `modelLoadFailed`; conform to `LocalizedError`), new `Sources/ThreeFingerSwitcher/AI/AIError.swift` (the central `AIError.message(for:)` translator + `AIPresentedError(headline, details)`).
- **Modified (boundary mapping):** `Sources/GemmaRuntime/GemmaMLXRuntime.swift` (`prepare` maps `Gemma4DownloadError`/load failures into `RuntimeError`, stops re-throwing raw; keeps the diagnostic log), optionally `Sources/GemmaRuntime/GemmaResumableDownloader.swift` (classify offline at the throw site).
- **Modified (state + de-leak):** `Sources/ThreeFingerSwitcher/AI/ModelManager.swift` (replace `\(error)` at `:170`/`:234` with the translator's headline; add the missing generic catch on the byte path at `:183`; stash `details`), `Sources/ThreeFingerSwitcher/AI/AICommandExecutor.swift` (route `message(for:)` through the central translator; fix the `?? "\(error)"` fallback).
- **Modified (tasks/selection honesty):** `Sources/ThreeFingerSwitcher/AI/Tasks/TaskSinks.swift` and `TaskDispatcher.swift` (wrap `DiskProjectStore` IO in `TaskError`; stop swallowing `NSWorkspace.open`/`Process`/URL-scheme failures), `Sources/ThreeFingerSwitcher/AI/SelectionService.swift` (surface `permissionDenied`; make `replaceSelection`/`pasteAtCursor` report real success).
- **Modified (UI):** `Sources/ThreeFingerSwitcher/Settings/ModelManagementView.swift` (concise headline + capped Text + details disclosure + Copy), `Sources/ThreeFingerSwitcher/Settings/SettingsView.swift` (scroll-safe Form), `Sources/ThreeFingerSwitcher/App/AppCoordinator.swift` (de-modal the download error surface; consume the translator), `Sources/ThreeFingerSwitcher/Overlay/AICommandCanvasView.swift` (length-cap parity).
- **No new dependencies, no new permissions.** Vendored `Gemma4DownloadError` is not edited (it lives in `.build/checkouts`); it is mapped at the runtime boundary. Builds remain `swift build`/`swift test` for Core and `xcodebuild` for the MLX target.
- **Depends on:** the existing `ai-command-band` feature code (this hardens it). It does not require `ai-command-band` to be archived first, but folds naturally alongside it.
